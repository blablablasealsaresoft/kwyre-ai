One-Click Installers

Install in under a minute

Platform-native installers handle everything: dependencies, firewall rules, system services, and model downloads.

🖥

Windows

Inno Setup · .exe installer

PowerShell installer configures Windows Firewall, creates Start Menu shortcut, and sets up the inference server as a background service.

powershell -ExecutionPolicy Bypass ` -File installer\install_windows.ps1
  • Windows Firewall rules (L2 enforcement)
  • Start Menu shortcut
  • Auto-detect NVIDIA GPU
🐧

Linux

Ubuntu/Debian · .deb + AppImage

Installs to /opt/kwyre, configures systemd service, sets up iptables rules for process-level network lockdown, creates dedicated kwyre user.

sudo bash installer/install_linux.sh sudo systemctl start kwyre
  • iptables rules (L2 kernel enforcement)
  • systemd service management
  • Dedicated kwyre system user
🍎

macOS

.pkg installer · Apple Silicon native

Installs to /opt/kwyre, configures launchd service, sets up PF firewall rules. Native MLX backend support for M1/M2/M3/M4 chips.

sudo bash installer/install_macos.sh sudo launchctl start com.kwyre.ai.server
  • PF firewall rules (L2 enforcement)
  • launchd service management
  • Apple Silicon / MLX auto-detect

Deployment Options

Four ways to deploy

Choose the deployment method that fits your environment and compliance requirements.

Docker Compose

Recommended for isolated deployments

Non-root container with dedicated kwyre user. Port mapping restricts to 127.0.0.1:8000 on host. Models auto-download on first run.

git clone https://github.com/blablablasealsaresoft/kwyre-ai cd kwyre-ai && cp .env.example .env docker compose up

Direct Python

For development and customization

Install inference-only dependencies. Place pre-quantized models in dist/. Full access to all source code for audit.

pip install -r requirements-inference.txt python server/serve_local_4bit.py

Kwyre Air (CPU)

No GPU required — any hardware

Uses llama.cpp via llama-cpp-python. Convert models to GGUF format or use pre-built GGUF from kwyre.com. Same API, same security stack.

KWYRE_GGUF_PATH=./models/kwyre-4b.gguf \ python server/serve_cpu.py

MLX (Apple Silicon)

Native M1/M2/M3/M4 acceleration

Uses mlx-lm for Metal-optimized inference. Convert models to MLX format with included converter. Unified memory architecture.

python model/convert_mlx.py \ --model Qwen/Qwen3-4B \ --output ./models/kwyre-4b-mlx python server/serve_mlx.py
Build Pipeline

Nuitka-compiled binary distribution

Server code compiles to standalone executable. No Python source distributed to customers. Source protection for your AI code.

What gets compiled vs. what stays as data

Nuitka compiles Python to C and builds a standalone executable. Data files are bundled alongside.

Compiled (Protected)

  • serve_local_4bit.py — inference server, API, security
  • tools.py — external API tool router
  • verify_deps.py — L3 dependency integrity
  • license.py — Ed25519 license validation
  • spike_serve.py — SpikeServe encoding

Data (Not Compiled)

  • chat/*.html — frontend UI (served as-is)
  • docs/ — compliance documentation
  • .env.example — configuration template
  • Model weights (.safetensors) — loaded at runtime
pip install nuitka ordered-set zstandard python build.py all # Compile + package + installer python build.py compile # Nuitka → kwyre-server[.exe] python build.py package # Stage data files python build.py installer # Platform installer python build.py clean # Clean artifacts
