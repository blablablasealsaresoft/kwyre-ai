Engine Architecture

Every optimization in the stack

Dual compression (weights + activations), speculative decoding, KV cache persistence, and real-time streaming.

Speculative Decoding

Qwen3-0.6B draft model generates candidate tokens. Qwen3-4B main model validates N tokens in a single forward pass. 2-3x throughput increase with identical output quality.

SpikeServe Activation Encoding

Dynamic spike encoding on the draft model only (84 MLP layers). Main model runs at full fidelity for accurate speculative validation. Dual compression: weights AND activations.

Spike QAT Training

Quantization-Aware Training with Straight-Through Estimator spike encoding. k-curriculum annealing from k=50 to k=5. The model learns to tolerate activation sparsity during training.

SSE Streaming

Token-by-token output via Server-Sent Events. Set stream: true in request body. TextIteratorStreamer delivers tokens as they generate, not after the full response completes.

KV Cache Persistence

Per-session cache stores past_key_values so follow-up messages skip re-encoding prior conversation. LRU eviction with configurable VRAM cap (default 2 GB). Cleared on session end or server shutdown.

Inference Queue

Serialized GPU access via single-worker queue. HTTP threads stay responsive during generation. Proper concurrency handling for multi-user mode — no race conditions, no CUDA deadlocks.

4-bit NF4 Quantization

Pre-quantized NF4 via bitsandbytes with double quantization. Both models fit in 3.9 GB VRAM. No on-the-fly quantization — clients download pre-compressed weights.

AWQ Quantization

Activation-Aware Weight Quantization for 1.4x faster inference when pre-quantized. Set KWYRE_QUANT=awq and point to your AWQ model. Alternative to NF4 for maximum throughput.

OpenAI-Compatible API

POST /v1/chat/completions drop-in replacement. Works with any OpenAI SDK client. Supports streaming and blocking modes, session management, and compliance endpoints.

Multi-Backend Architecture

Three backends, one security core

All backends share security_core.py — identical 6-layer stack, identical API surface, identical HTML frontend.

GPU Backend

serve_local_4bit.py · NVIDIA CUDA

Full-featured inference with NF4/AWQ quantization, speculative decoding, SpikeServe activation encoding, SSE streaming, KV cache persistence, and inference queue.

  • QuantizationNF4 / AWQ
  • SpeculativeYes (0.6B draft)
  • SpikeServeDraft model
  • StreamingSSE

CPU Backend (Kwyre Air)

serve_cpu.py · llama.cpp · GGUF

Run on any hardware without a GPU. Uses llama-cpp-python for GGUF model inference. Same API surface. Same security stack. Convert models with model/convert_gguf.py.

  • FormatGGUF
  • SpeculativeNo
  • StreamingSSE
  • GPU RequiredNone

MLX Backend

serve_mlx.py · Apple Silicon · Metal

Native Apple Silicon acceleration for M1, M2, M3, and M4 chips. Uses mlx-lm for Metal-optimized inference. Convert models with model/convert_mlx.py.

  • FormatMLX
  • SpeculativeNo
  • StreamingSSE
  • GPU RequiredNone (M-chip)
